What changed vs the app you have. The mobile app is already strong (offline PowerSync, jobs, timers, quotes, invoices, Tap-to-Pay, auto travel-log + geofence check-in) — but it has no inbox, no messaging and no ETA, and the web app already ships all of that. This overhaul brings it to mobile by reusing the web endpoints: /api/messages/*, /api/sms/send, /api/notify, plus a new /api/sms/eta.

New on mobile: a first-class Inbox tab (enquiries + SMS + bookings + unmatched, mirroring web /messages); one-touch "On my way" using the GPS/LocationIQ/site-coords/geofence already in the app; a redesigned job screen with a quick-action row, status stepper and the Forms the app is named for. SMS is shown dark-aware — the reply UI is built but disabled until Twilio is switched on, exactly like the web. Full rationale, file-by-file plan and phased roadmap are in the accompanying Word brief.
